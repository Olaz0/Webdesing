    • Project - Restauracja C/C++

    Project description

    Our project is a program/code that emulates the cash register in a restaurant. The purpose of this solution is to enable the restaurant to simulate operations related to handling orders, generating receipts and storing order data in a database.

    Screen shots

    Functions

    • Restaurant menu: The checkout emulator has defined different types of food and drinks that can be ordered. This menu usually includes a wide variety of dishes such as starters, main dishes, desserts, drinks, etc. Each dish is represented by a name, description, price and other characteristics that may be relevant to the customer.
    • Ordering: The program allows the user to place an order by selecting the appropriate items from the menu. This can be done through the user interface, where the customer can browse the available dishes and add them to the order. The system allows you to add many items to one order.
    • Receipt generation: After placing an order, the program generates a receipt, which contains a description of the order, a list of ordered dishes and drinks, the total amount to be paid and the date of the order. The receipt can be displayed on the screen or printed in paper form.
    • Receipt database: All orders placed are stored in the database. Each receipt is represented as an entry in the database, containing information such as the receipt number, list of dishes ordered, date of order, etc. The database allows you to store and manage order data, which allows the restaurant to view order history and analyze the data later.
    • Management of the menu: In order to adapt the menu to the changing preferences of customers, the program allows you to manage the restaurant's menu. The restaurant owner can add, remove or edit existing menu items such as dishes, drinks, prices, etc.
